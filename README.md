# TODO GitOps Repository
This repository contains the GitOps definition of the TODO environment.

## Usage
Modify the `app.yaml` file and create a pull request (PR). When the change is merged GitHub
will execute the deployment workflow to apply the change to the target environment.

> Updates made to non-main branches (feature or PR) will invoke the build action, however rather than
> deploying the changes the action will validate the update and generate a difference report (delta)
> showing the set of computed changes.
